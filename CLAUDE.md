# AmsalGame_proto — 프로젝트 컨텍스트

## 프로젝트 개요
탑뷰 암살 멀티플레이어 게임
- Unity 6.3 LTS, URP, 3D Core
- IDE: VSCode + Claude Code (에이전트)
- 언어: C#
- 버전관리: Git / GitHub
- 출시 목표: Steam (Windows)

## 핵심 철학
"보이지 않는 것이 더 무섭다"
- 사운드가 핵심 게임플레이 메커닉
- 항상 움직여야 하는 긴장감 유지
- 암살 우선, 불가피할 때 CQC
- 개인 행동과 정보전 중심

### 추가 작업 (Phase 1 이후 보강)
- 맵 확장: `Tools → AmsalGame → 맵 확장 (방 2개 + 복도 추가)` — 기존 방(57.8m×57.8m 실측)에서 벽 2개를 문 있는 벽으로 교체하고 복도(폭 3m)로 연결된 Room B/Room C(7×7m)를 절차적으로 생성. 방 크기는 Player 위치에서 Raycast로 실측(메시 bounds는 신뢰 불가 — 아래 기록 참고). `Phase1SceneSetup.cs`의 `ExpandMap()`.
- 적 앞/뒤 색상 마커: Enemy 캡슐 자식으로 빨강(정면)/초록(후방, 암살 가능 구간) 구슬 부착 — `SetupFacingMarkers()`.
- AI 전술 행동 (플레이테스트 피드백 "따라오기만 함" 대응): ① 군집 방지 — `AIController.All` 정적 레지스트리 기반, 순찰 목적지에서 아군 4m 이내 제외 + 이동 중 주기적 분리. ② Chase 협공 분산 — 아군이 이미 Chase/Combat이면 110~150° 측면 우회(InstanceID 타이브레이커, 우회 중 시야각 체크 생략). ③ 코너 체크 — Suspicious 한정, `path.corners` 급커브(65°+) 접근 시 잠깐 멈춰 확인. ④ CQC 3지선다 — 공격 45%/후퇴 30%/막기 나머지, 후퇴는 백스텝하며 플레이어 주시(`updateRotation` Combat에서만 수동), 사거리 밖 공격은 막기로 대체. 전부 플레이테스트 검증 전.
- 미니맵 아키텍처 기록: 씬 조명이 어두워 실제 지오메트리 렌더 불가 → 전용 카메라가 Minimap 레이어의 언릿 클론만 렌더. 클론은 `RebuildMinimapGeometry()`가 delete-and-rebuild (맵 확장/미로화 메뉴가 자동 갱신 호출). 핑 머티리얼은 URP/Unlit 투명 대신 `Sprites/Default`(블렌드 키워드 함정 회피). 사망 후 유령 핑은 근본 수정됨 — `PlayerController.OnDisable`이 이동 상태 리셋, `OnGameEnded` 게이트는 이중 안전장치로 유지.
- 플레이어 사망 처리: `CombatController.HandleDeath`가 플레이어 계열 컴포넌트(PlayerController/PlayerCombatInput/RangedWeaponController/SoundEmitter/ExposureSystem)를 일괄 정지 — 시체가 발소리/노출 경고/공격 입력/무기 사용을 계속 내던 버그 수정. `ExposureSystem.OnDisable`이 경고/노출 잔여 상태도 정리(노출 중이었으면 OnExposureEnd 발행해 오버레이/미니맵 링 해제).
- 미니맵 핑 스로틀 (플레이테스트 피드백 "핑이 너무 촘촘함" 대응): `MinimapController`가 소리 주체별로 최소 1.2초 간격으로만 핑을 그림 — 발소리는 0.3초마다 emit되지만 표시만 걸러냄 (AI 청각 게임플레이는 그대로). 총성 등 전역 경보(반경 50m+)는 스로틀 무시하고 항상 표시.
- 특수부대풍 암전 조명 (플레이테스트 피드백 "제한된 시야, 밀실 느낌" 대응): `Tools → AmsalGame → 특수부대 조명 적용 (제한 시야 강화)` — 태양광 0.04(사실상 소등) + 환경광 Flat 저조도(스카이박스 기여 차단) + 메인 카메라 배경 SolidColor 검정 + 시야 SpotLight 100°/14m/강도5 → 70°/11m/강도7(냉백색) + 근접 PointLight(반경 3.5m, 강도 1.2) 추가로 발밑 최소 가시성 확보. `ApplyTacticalLighting()`이 값을 항상 덮어쓰는 방식이라 수치 튜닝 후 메뉴 재실행으로 반영 가능. Phase 1 자동 구성(`SetupVisionLight`)도 같은 함수를 쓰므로 새 씬에도 동일 적용.
- 시야 마스크 (유저 요청 "시야 밖은 검은 블러/매우 흐림 처리"): `VisionMask.shader` + `VisionMaskOverlay.cs` — 메인 카메라 근평면 앞 프러스텀 크기 쿼드가 URP 오파크 텍스처를 샘플해, 픽셀 luminance가 낮으면(손전등 밖) 12탭 디스크 블러 + 탈색 + 암전, 높으면 원본 유지. 손전등 그림자가 이미 벽 차폐를 계산하므로 별도 시야 판정 불필요 — "밝음=보임"을 마스크 기준으로 재사용하는 구조. 이를 위해 환경광을 완전 검정 대신 (0.07, 0.08, 0.10) 저조도로 유지 (시야 밖이 블러 속 실루엣으로 보이게; 환경광 luminance는 마스크 임계 `_VisibleLumMin` 0.12보다 낮아야 은폐 유지). 튜닝은 `Assets/Materials/VisionMask.mat` 인스펙터(임계 밝기/블러 반경/은폐 밝기·탈색·틴트), 배선은 특수부대 조명 메뉴가 담당(`SetupVisionMask()`). 효과 끄기 = 메인 카메라의 VisionMaskOverlay 비활성화. 제약: ① URP Opaque Texture 필수(현재 켜져 있음, 메뉴가 안전장치로 재확인) ② 메인 카메라의 반투명 오브젝트는 오파크 텍스처에 없어 마스크에 덮임(현재 반투명은 미니맵 레이어 전용이라 무영향 — 파티클 추가 시 주의) ③ Claude Code가 셰이더 컴파일을 직접 검증 못 함 — 화면이 마젠타면 셰이더 에러이니 Console 확인.

### 🔲 미착수 (Phase 1 잔여)
- 총알 별도 파밍 (현재는 권총 파밍 시 탄약도 함께 획득하는 것으로 단순화함)
- FOV 라이팅 수치 튜닝 (특수부대풍 암전으로 1차 개편했으나 여전히 시각 확인 전 추정치 — 메뉴 재실행으로 튜닝 가능)
- 군집 레이더 노출 미니맵 표시 (싱글플레이에서 군집 페널티가 트리거 불가라 Phase 3으로 이연)
- AI 발소리 없음 — 미니맵 핑이 현재 플레이어 계열 소리만 표시 (AI에 SoundEmitter 계열 부착은 Phase 2+)

### ⚠️ 알아두면 좋은 사실
- `Assets/Scenes/SampleScene.unity`가 세션 도중 텍스트(YAML)에서 Unity 바이너리 직렬화로 바뀜 — Claude Code가 더 이상 grep으로 씬을 못 읽음. Editor 자동화 스크립트가 Unity API(GameObject.Find, Renderer.bounds, Physics.Raycast)로 값을 읽는 방식이라 기능엔 문제 없지만, 앞으로 씬 관련 문의 시 텍스트 파싱이 아니라 Editor 스크립트로 조회해야 함.
- ProBuilder로 만든 벽/바닥의 `Renderer.bounds`는 신뢰 금지 — 한 번 실제 크기(900+ 단위)로 완전히 잘못된 값을 반환한 적 있음. 크기가 중요한 배치는 `Physics.Raycast`로 실측할 것 (`ExpandMap`의 `MeasureHalfExtent` 참고).

## 입력 시스템
Unity Input System (새 방식) 사용 — `InputSystem_Actions` 자동생성 클래스

| 액션 | 실제 키 | 게임 내 역할 |
|------|---------|-------------|
| Move | WASD | 이동 방향 |
| Sprint | Left Shift | **걷기 모드** (속도 감소, 발소리 최소화) |
| Attack | LMB | 컨텍스트 감지형 — 암살 가능하면 암살, 아니면 CQC 공격 |
| (직접 폴링) | RMB | 막기(홀드) / 누른 직후 짧은 구간 패링 |
| (직접 폴링) | F | CQC 그랩 (막기 중인 상대에게만 유효) |
| (직접 폴링) | Q | 칼 던지기 (마우스 방향) |
| (직접 폴링) | R | 던진 칼 회수 (1.5m 이내) |
| (직접 폴링) | MMB | 권총 발사 (파밍 후 1회, 소모성) |

> Sprint 액션을 "걷기"로 역용 — Shift를 누르면 느려지는 설계
> RMB/F/Q/R/MMB는 InputSystem_Actions 액션으로 등록되어 있지 않아 `Mouse.current`/`Keyboard.current`로 직접 폴링 (`PlayerCombatInput.cs`, `RangedWeaponController.cs`). 추후 액션 에셋 정리 시 정식 액션으로 승격 검토 — 특히 콘솔 대응 시 게임패드 바인딩이 없는 입력들이라 우선순위 높음.

## 이동 시스템
| 상태 | 속도 | 발소리 반경 |
|------|------|-------------|
| 이동 | 요원별 고유 속도 | 중간 |
| 걷기 (Shift) | 기본의 50% (AgentData 기본값) | 최소 |
| 웅크리기 | 추후 검토 | — |

**AgentData 기본값 (ScriptableObject)**
| 필드 | 기본값 | 설명 |
|------|--------|------|
| moveSpeed | 3.5f | 기본 이동 속도 (m/s) |
| walkSpeedMultiplier | 0.5f | 걷기 속도 배율 |
| moveSoundRadius | 8f | 이동 발소리 반경 (m) |
| walkSoundRadius | 3f | 걷기 발소리 반경 (m) |

PlayerController에 AgentData가 없을 때 폴백: 속도 3.5f, 걷기 배율 0.6f

## 전투 시스템

### 암살 (주력) ✅ 구현됨
- 적이 나를 인식하지 못한 상태에서 후방 접근 시 발동
- 즉사, 무음 처리
- **입력: LMB** (컨텍스트 감지 — `AssassinationSystem.CanAssassinate`가 true면 암살, 아니면 CQC 공격으로 분기. `PlayerCombatInput.cs`)
- 암살 가능 거리: 1.5f
- 암살 가능 각도: 적 후방 120도 이내
- AI가 Chase/Combat 상태일 때 암살 불가 (인식 여부 판정)
- 암살 가능 시 적 머리 위에 월드스페이스 인디케이터 표시 (`AssassinationIndicatorUI.cs`)

### CQC — 근접 전투 ✅ 구현됨 (`CombatController.cs`)
발동 조건: AI가 Chase 상태에서 근접 거리(기본 2m)로 진입 시 자동으로 Combat 상태 전환 → 서로 교전

| 입력 | 행동 | 설명 |
|------|------|------|
| LMB | 공격 | 기본 타격 (2 dmg, 막기 중 상대는 1 dmg) | 
| RMB | 막기 | 피해 감소, 막기 중엔 본인 공격/그랩 불가 |
| RMB 타이밍 | 패링 | 막기 시작 직후 0.3초 이내 피격 시 상대 경직 |
| F키 | 잡기 | 막기 중인 상대에게 유효, 즉시 처치 |

CQC 흐름:
- 패링 성공 → 상대 경직(`IsStaggered`) → 이후 LMB로 암살 피니시 가능 (즉사)
- 기본 공격 히트 → 체력 감소
- 잡기 성공 → 상대 무력화 → 즉시 처치

AI 측 CQC 행동은 `AIController.DecideCombatAction()`의 단순 확률 기반 플레이스홀더(60% 공격 / 40% 막기, 0.8~1.6초 간격) — 실제 전투 AI 튜닝은 밸런싱 예정.

### 원거리 (극히 제한적) ✅ 구현됨 (`RangedWeaponController.cs`, `ThrownKnife.cs`, `WeaponPickup.cs`)

**칼 던지기 (기본 보유)**
- Q: 마우스 방향 발사, R: 1.5m 이내 접근 시 회수
- 무음, 빗나가면 착지음(반경 6m) 발생 → 주변 AI/플레이어 반응
- 6초 내 회수 못하면 자동 소멸 (안전장치, 스펙 외 추가 구현)

**권총 (파밍 무기)**
- 시작 시 미보유, A/B 사이트(`SetupSites`로 배치)에서 파밍해야 사용 가능
- 스폰: 맵 내 A/B 2개 사이트 각 1자루, 총 2자루 제한
- ~~총알 별도 스폰~~ → 파밍 시 탄약도 함께 획득하는 것으로 단순화 (1발 소모성 무기라 별도 탄약 엔티티의 실익이 적어 스코프 축소)
- 사용(MMB): 1발 발사 후 소모, 벽/적 판정 레이캐스트(사거리 30m)
- 성능: 명중 시 즉사
- 리스크: 발사 시 맵 전체 사운드 → 위치 노출 (반경 사실상 무한대로 처리)
- 라운드 리셋 시 재스폰: 미구현 (라운드 시스템 자체가 없음 — Phase 2 이후 과제)
- 게임플레이 의도: A/B 사이트 거점 경쟁 + 위험 동선 트레이드오프

## 요원 시스템
모든 요원 동일 외형, 유틸리티 스킬로 차별화
AgentType enum: `Offensive` / `Defensive` / `Neutral`

### 🛡️ 방어형 (Defensive)
이동속도: 느린 편
맵 통제 + 정보 수집 + 함정 설치 특화
존버 패널티 정지 허용 시간: 기본 +2~3초 (함정 설치 고려)

| 스킬 | 설명 |
|------|------|
| 소리 트랩 | 적이 밟으면 소리 이벤트 발생, 위치 노출 |
| 부비트랩 | 적이 밟으면 피해 + 경직, CQC 기회 창출 |
| 침묵 구역 | 반경 내 발소리 차단, 아군 은신 지원 |
| 페이크 사운드 | 다른 위치에서 발소리 발생, 적 유인 |

### ⚔️ 공격형 (Offensive)
이동속도: 빠른 편
기동력 + 암살 기회 창출 특화
존버 패널티 정지 허용 시간: 기본과 동일

| 스킬 | 설명 |
|------|------|
| 대시 | 짧은 거리 순간이동, 발소리 최소화 |
| 집중 | 이동속도 대폭 증가, 발소리 소폭만 증가 |
| 무음 질주 | 빠르게 이동하지만 발소리 반경 대폭 감소 |
| 마킹 | 벽 너머 적 위치를 짧은 시간 파악 |

### 팀 조합 시너지
- 방어형이 함정으로 적 위치 파악 → 공격형이 빠르게 암살
- 공격형이 교란 → 방어형이 도주로에 트랩 설치
- 팀원은 최대 2명까지만 같이 행동 권장 (군집 페널티 참고)

## 군집 페널티 시스템 ✅ 구현됨 (`TeamMember.cs`, `ClusterPenaltySystem.cs`)
아군끼리 뭉치는 플레이 억제 → 개인 행동 유도

> 현재 싱글플레이 프로토타입에는 `TeamMember`를 가진 개체가 Player 1명뿐이라 3인 집결 조건이 실전에서 트리거되지 않음. 감지 로직(발소리 배율, CQC 피해 배율, 암살 무효)은 이미 `SoundEmitter`/`CombatController`/`AssassinationSystem`에 연결돼 있어 Phase 3에서 팀원이 추가되면 바로 검증 가능. 레이더 노출은 미니맵은 생겼지만(적팀 관점이 없는 싱글플레이라 의미 없음) Phase 3으로 이연.

### 2명까지 — 정상
- 암살 판정 정상
- 발소리 정상
- CQC 피해 정상

### 3명 이상 동일 반경 내 집결 시 — 페널티 발동

| 페널티 | 효과 |
|--------|------|
| 레이더 노출 | 적 미니맵에 위치 실시간 노출 |
| 발소리 누적 | 인원수 × 1.5배 반경 증가 |
| CQC 피해 증가 | 근접 전투 시 받는 피해 증가 (밸런싱 예정) |
| 암살 무효 | 즉사 판정 → CQC 강제 전환 |

### 설계 의도
- 뭉치면 안전해 보이지만 실제론 더 위험한 구조
- 암살 게임 컨셉 유지 → 개인 행동, 정보전 강제
- 팀원 2명까지는 협력 가능 (완전 고립 방지)

### 밸런싱 예정
- 군집 감지 반경 수치
- CQC 피해 증가율
- 레이더 노출 주기/지속 시간

## 존버 방지 시스템 (노출 페널티) ✅ 구현됨 (`ExposureSystem.cs`)
플레이어가 일정 시간 정지 시 맵에 위치 노출. 노출은 한 번 시작되면 이동해도 정해진 시간만큼 유지(이동으로 페널티 즉시 회피 방지), 경고 단계는 이동 즉시 초기화.

| 항목 | 현재 기본값 (밸런싱 예정) |
|------|----|
| 정지 허용 시간 | 6초 |
| 경고 카운트다운 | 2.5초 |
| 노출 지속 시간 | 1~3초 랜덤 |
| 초기화 조건 | 이동 시작 즉시 (경고 단계만 — 노출 발동 후엔 유지) |
| 방어형 요원 | 정지 허용 시간 +2.5초 |

플레이어 본인: 우상단 게이지(`ExposureGaugeUI.cs`, 경고 진행률, R6풍 어두운 슬롯 배경 + 옐로 필) + 노출 시 화면 전체 빨간 오버레이로 근사 구현 (스펙상 "가장자리" 비네트는 추후 비주얼 폴리싱 필요)
본인 미니맵: 노출 중 플레이어 마커 주위 빨간 펄스 링 표시 (`MinimapController`). 적팀 미니맵 실루엣은 싱글플레이라 대상 없음 — Phase 3

사운드 시스템과의 시너지:
- 가만히 있으면 위치 노출
- 함부로 뛰면 발소리로 위치 노출
- 걷기의 전략적 가치 극대화

## HUD 구성
| 위치 | 요소 |
|------|------|
| 좌상단 | 미니맵 (사운드 핑, 레이더 노출 표시) |
| 우상단 | 존버 방지 게이지 + 경고 |
| 좌하단 | 요원 정보 (타입 / HP 숫자+바 / 스킬) |
| 우하단 | 무기 (아이콘 + 상태 표시) |

R6(레인보우식스)풍 스타일 적용 현황: 미니맵은 원형 마스크 + 외곽 링, 존버 게이지는 어두운 슬롯 배경 + 옐로 필/텍스트로 반영됨. `SetupHud`가 존버 게이지 클러스터를 delete-and-rebuild 하므로 스타일 변경 후 메뉴 재실행으로 갱신 가능 (암살 인디케이터는 유지 방식). 좌하단 요원 정보/우하단 무기 HUD는 Phase 2.

시야(FOV): ✅ 구현됨 — 특수부대풍 암전 + luminance 기반 시야 마스크
- Player 자식에 SpotLight "손전등" (각도 70°, 사거리 11m, 강도 7, 냉백색, 실시간 그림자) → 벽에 막히면 자연스럽게 차폐, 이게 사실상 유일한 시야
- 태양광 0.04(사실상 소등) + 환경광 Flat 저조도(0.07, 0.08, 0.10) + 메인 카메라 배경 검정 — 밀실/작전 느낌
- 근접 PointLight(반경 3.5m, 강도 1.2)로 발밑 최소 가시성만 확보 (조작 불능 방지용, 정보 가치는 거의 없음)
- 시야 마스크 (`VisionMask.shader` + `VisionMaskOverlay.cs`): 화면에서 어두운 픽셀(손전등 밖)은 검은 블러 + 탈색으로 뭉개고 밝은 픽셀만 선명하게 — "시야가 제한되어 있다"를 시각적으로 명시. 스텐실/렌더러 피처 없이 오파크 텍스처를 샘플하는 쿼드 1장이라 구조 단순
- 배선: `ApplyTacticalLighting()` — `Tools → AmsalGame → 특수부대 조명 적용 (제한 시야 강화)` 메뉴로 언제든 재적용/튜닝 (항상 값 덮어쓰기, 마스크 수치는 VisionMask.mat)
- 조명 수치·마스크 임계는 여전히 시각 확인 전 추정치라 실제로 봤을 때 튜닝 필요할 가능성 높음

- 암살 가능 거리/각도
- 발소리 반경 수치

- AI CQC 행동 패턴 (현재 60% 공격 · 40% 막기 확률 기반 플레이스홀더 — 실제 전투 AI로 교체 필요)
- 군집 감지 반경 (현재 5m) / 군집 인원 임계값 (현재 3명)

- 레이더 노출 주기/지속 시간 (Phase 3 이연 — 싱글플레이라 적팀 미니맵 없음)
- 미니맵 수치: 레이더 범위(18m)/핑 지속(1초)/핑 클램프(1.5~25m)/총성 임계(50m)/주체별 핑 스로틀(1.2초)/색상 — 전부 1차 추정치
- FOV 손전등 SpotLight 밝기(7)/각도(70°)/사거리(11m), 태양광(0.04)/환경광(0.07~0.10)/근접 PointLight(3.5m, 1.2) — 특수부대풍 암전 1차 수치, 시각 확인 전
- 시야 마스크 수치 (`VisionMask.mat`): 은폐 임계 luminance(0.12~0.30)/블러 반경(12px)/은폐 밝기 배율(0.5)/탈색(0.8)/틴트 — 전부 시각 확인 전 추정치. 환경광을 올리면 임계도 같이 올려야 은폐 유지됨
- 칼 던지기 속도(18m/s)/회수 거리(1.5m)/자동 소멸 시간(6초)
- 권총 사거리(30m)
- A/B 사이트 위치 (Player 스폰 기준 상대 오프셋으로 배치 — 실제 맵 형태 확인 후 재배치 필요할 수 있음)
