# AmsalGame_proto — 프로젝트 컨텍스트

## 프로젝트 개요
탑뷰 암살 멀티플레이어 게임
- Unity 6.3 LTS, URP, 3D Core
- IDE: VSCode + Claude Code (에이전트)
- 언어: C#
- 버전관리: Git / GitHub
- 출시 목표: Steam (Windows)

## 핵심 철학
"보이지 않는 것이 더 무섭다"
- 사운드가 핵심 게임플레이 메커닉
- 항상 움직여야 하는 긴장감 유지
- 암살 우선, 불가피할 때 CQC
- 개인 행동과 정보전 중심

## 입력 시스템
Unity Input System (새 방식) 사용 — `InputSystem_Actions` 자동생성 클래스

| 액션 | 실제 키 | 게임 내 역할 |
|------|---------|-------------|
| Move | WASD | 이동 방향 |
| Sprint | Left Shift | **걷기 모드** (속도 감소, 발소리 최소화) |
| F키 | F | 암살 실행 (AssassinationSystem) |

> Sprint 액션을 "걷기"로 역용 — Shift를 누르면 느려지는 설계

## 이동 시스템
| 상태 | 속도 | 발소리 반경 |
|------|------|-------------|
| 이동 | 요원별 고유 속도 | 중간 |
| 걷기 (Shift) | 기본의 50% (AgentData 기본값) | 최소 |
| 웅크리기 | 추후 검토 | — |

**AgentData 기본값 (ScriptableObject)**
| 필드 | 기본값 | 설명 |
|------|--------|------|
| moveSpeed | 3.5f | 기본 이동 속도 (m/s) |
| walkSpeedMultiplier | 0.5f | 걷기 속도 배율 |
| moveSoundRadius | 8f | 이동 발소리 반경 (m) |
| walkSoundRadius | 3f | 걷기 발소리 반경 (m) |

PlayerController에 AgentData가 없을 때 폴백: 속도 3.5f, 걷기 배율 0.6f

## 사운드 메커닉 (핵심 게임플레이)
| 행동 | 소리 범위 |
|------|-----------|
| 이동 | 중간 반경 |
| 걷기 | 최소 반경 |
| 총소리 | 맵 전체 + 위치 노출 |
| 칼 던지기 | 무음 (빗나가면 착지음) |
| 근접 암살 | 무음 |
| 소리 트랩 발동 | 트랩 반경 내 전파 |

사운드는 메인 화면이 아닌 좌상단 미니맵에 핑으로 표시
- 핑 크기 = 소리 거리/크기 비례
- 잠깐 떴다 사라지는 방식

## 전투 시스템

### 암살 (주력)
- 적이 나를 인식하지 못한 상태에서 후방 접근 시 발동
- 즉사, 무음 처리
- **입력: F키**
- 암살 가능 거리: 1.5f
- 암살 가능 각도: 적 후방 120도 이내
- AI가 Chase 상태일 때 암살 불가 (인식 여부 판정)

### CQC — 근접 전투
발동 조건: 서로 시야에서 인식한 상태로 근접 거리 진입 시

| 입력 | 행동 | 설명 |
|------|------|------|
| LMB | 공격 | 기본 타격 | 
| RMB | 막기 | 피해 감소 |
| RMB 타이밍 | 패링 | 상대 경직 + 반격 기회 |
| F키 | 잡기 | 막기 중인 상대에게 유효 |

CQC 흐름:
- 패링 성공 → 상대 경직 → 암살 피니시 가능
- 강공격 히트 → 체력 감소
- 잡기 성공 → 상대 무력화 → 처치

### 원거리 (극히 제한적)

**칼 던지기 (기본 보유)**
- 마우스 방향 발사, 회수 필요
- 무음, 빗나가면 착지음 발생 → 주변 AI/플레이어 반응

**권총 (파밍 무기)**
- 시작 시 미보유, 맵에서 파밍해야 사용 가능
- 스폰: 맵 내 A/B 2개 사이트 각 1자루, 총 2자루 제한
- 총알: 권총 근처에 같이 스폰 (별도 획득)
- 사용: 1발 발사 후 버려짐 (소모성)
- 성능: OP급 위력, 신중하게 사용해야 함
- 리스크: 발사 시 맵 전체 사운드 → 위치 노출
- 라운드 리셋 시: 맵에 다시 스폰으로 초기화
- 게임플레이 의도: A/B 사이트 거점 경쟁 + 위험 동선 트레이드오프

## 요원 시스템
모든 요원 동일 외형, 유틸리티 스킬로 차별화
AgentType enum: `Offensive` / `Defensive` / `Neutral`

### 🛡️ 방어형 (Defensive)
이동속도: 느린 편
맵 통제 + 정보 수집 + 함정 설치 특화
존버 패널티 정지 허용 시간: 기본 +2~3초 (함정 설치 고려)

| 스킬 | 설명 |
|------|------|
| 소리 트랩 | 적이 밟으면 소리 이벤트 발생, 위치 노출 |
| 부비트랩 | 적이 밟으면 피해 + 경직, CQC 기회 창출 |
| 침묵 구역 | 반경 내 발소리 차단, 아군 은신 지원 |
| 페이크 사운드 | 다른 위치에서 발소리 발생, 적 유인 |

### ⚔️ 공격형 (Offensive)
이동속도: 빠른 편
기동력 + 암살 기회 창출 특화
존버 패널티 정지 허용 시간: 기본과 동일

| 스킬 | 설명 |
|------|------|
| 대시 | 짧은 거리 순간이동, 발소리 최소화 |
| 집중 | 이동속도 대폭 증가, 발소리 소폭만 증가 |
| 무음 질주 | 빠르게 이동하지만 발소리 반경 대폭 감소 |
| 마킹 | 벽 너머 적 위치를 짧은 시간 파악 |

### 팀 조합 시너지
- 방어형이 함정으로 적 위치 파악 → 공격형이 빠르게 암살
- 공격형이 교란 → 방어형이 도주로에 트랩 설치
- 팀원은 최대 2명까지만 같이 행동 권장 (군집 페널티 참고)

## 군집 페널티 시스템
아군끼리 뭉치는 플레이 억제 → 개인 행동 유도

### 2명까지 — 정상
- 암살 판정 정상
- 발소리 정상
- CQC 피해 정상

### 3명 이상 동일 반경 내 집결 시 — 페널티 발동

| 페널티 | 효과 |
|--------|------|
| 레이더 노출 | 적 미니맵에 위치 실시간 노출 |
| 발소리 누적 | 인원수 × 1.5배 반경 증가 |
| CQC 피해 증가 | 근접 전투 시 받는 피해 증가 (밸런싱 예정) |
| 암살 무효 | 즉사 판정 → CQC 강제 전환 |

### 설계 의도
- 뭉치면 안전해 보이지만 실제론 더 위험한 구조
- 암살 게임 컨셉 유지 → 개인 행동, 정보전 강제
- 팀원 2명까지는 협력 가능 (완전 고립 방지)

### 밸런싱 예정
- 군집 감지 반경 수치
- CQC 피해 증가율
- 레이더 노출 주기/지속 시간

## 존버 방지 시스템 (노출 페널티)
플레이어가 일정 시간 정지 시 맵에 위치 노출

| 항목 | 값 |
|------|----|
| 정지 허용 시간 | 5~8초 (밸런싱 예정) |
| 경고 카운트다운 | 2~3초 |
| 노출 지속 시간 | 1~3초 랜덤 |
| 초기화 조건 | 이동 시작 즉시 |
| 방어형 요원 | 정지 허용 시간 +2~3초 |

플레이어 본인: 화면 가장자리 빨간 경고 + 카운트다운 (우상단 게이지)
적팀: 미니맵 위치 핑 + 실루엣 노출

사운드 시스템과의 시너지:
- 가만히 있으면 위치 노출
- 함부로 뛰면 발소리로 위치 노출
- 걷기의 전략적 가치 극대화

## AI 시스템
상태머신 (FSM) 기반, NavMeshAgent 사용

| 상태 | 설명 |
|------|------|
| Idle | 제자리 대기 (1~3초 랜덤) |
| Patrol | 스폰 기준 반경 10m 내 랜덤 웨이포인트 순찰 |
| Suspicious | 소리 감지 위치로 이동 후 5초 탐색 |
| Chase | 시야 내 플레이어 발견, NavMesh 추격 |
| Alert | 동료 사망 감지, 4초 경계 후 복귀 |

**AIController 기본값**
| 항목 | 값 |
|------|----|
| viewDistance | 8f |
| viewAngle | 90° |
| hearingRadius | 15f |
| patrolRadius | 10f |
| patrolSpeed | 2f |
| chaseSpeed | 4.5f |
| suspiciousDuration | 5f |
| alertDuration | 4f |

공개 메서드: `TriggerAlert()` — 동료 사망 시 외부에서 호출 (아직 연결 미완료)

### Phase 1 — 싱글 프로토타입 (현재 목표)
